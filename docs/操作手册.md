# Dify-KB-Eval 操作手册

## 1. 适用范围

本文面向研发、测试和交付前验证人员，用于启动并使用独立知识库评测系统。

评测系统目录为本仓库根目录，以下命令默认在 `Dify-KB-Eval/` 下执行。

注意：

- 这是独立评测工具，不是业务系统功能。
- 不需要修改其他业务前端或业务后端。
- 不需要执行其他业务系统的 Alembic migration。**生产部署必须在自己的 DB 上跑 `alembic upgrade head`**,然后启动后端时设 `RUN_DB_BOOTSTRAP=false`(见 `scripts/migrations/LEGACY.md`)。开发态保持 `RUN_DB_BOOTSTRAP=true` 即可。

## 2. 前置条件

本机需要具备：

- Python `>= 3.12`
- `uv`
- Node.js 和 npm
- Dify 服务已启动，默认示例 API 地址为 `http://localhost/v1`
- 目标知识库已经在 Dify 中完成索引
- 已准备 Dify Knowledge Base API Key

评测集目录：

```text
datasets/
datasets/generated/
```

## 3. 第一次安装

进入评测系统目录：

```powershell
cd <项目根目录>
```

安装后端依赖：

```powershell
uv sync
```

`MarkItDown` 已随项目的 `uv` 依赖一起安装，不需要再额外执行 `pip install`。

安装前端依赖：

```powershell
cd <项目根目录>\frontend
npm install
```

提交或交付前执行完整验证：

```powershell
cd <项目根目录>
.\verify.ps1
```

## 4. Mock 模式预览前端

Mock 模式不需要启动后端，也不需要 Dify，适合前端验收页面效果。

```powershell
cd <项目根目录>\frontend
$env:VITE_USE_MOCK="true"
npm run dev
```

浏览器访问：

```text
http://127.0.0.1:5598
```

开启 mock 后，页面可以查看：

- 评测配置表单
- 历史评测列表
- 运行详情页
- 指标卡片
- 失败样本
- Markdown 报告
- 产物下载按钮

## 5. 启动真实后端

打开一个新的 PowerShell 窗口：

```powershell
cd <项目根目录>
uv run uvicorn backend.app:app --reload --host 127.0.0.1 --port 8200
```

检查后端是否启动成功：

```powershell
Invoke-RestMethod http://127.0.0.1:8200/api/health
```

正常返回示例：

```json
{
  "status": "ok",
  "service": "dify-kb-eval",
  "version": "0.1.0"
}
```

查看评测集列表：

```powershell
Invoke-RestMethod http://127.0.0.1:8200/api/datasets
```

## 6. 启动真实前端

再打开一个 PowerShell 窗口：

```powershell
cd <项目根目录>\frontend
$env:VITE_API_BASE_URL="http://127.0.0.1:8200"
npm run dev
```

浏览器访问：

```text
http://127.0.0.1:5598
```

说明：

- 前端默认使用真实后端。
- `VITE_API_BASE_URL` 是 Vite 代理到评测后端的地址。
- 页面中填写的 Dify API 地址和 Key 会成对保存在后端数据库的历史连接配置中，用于刷新后自动回填和下拉复用；请只在可信本机或可信内网使用真实凭据。

## 7. 发起一次小样本评测

建议第一次只跑 3 条样本，先验证 Dify、知识库 ID 和报告链路。

页面填写建议：

| 字段 | 建议值 |
|---|---|
| Dify API 地址 | `http://localhost/v1` |
| Dify API Key | 必填，使用 Dify Knowledge Base API Key |
| 评测集 | `华为 S1720 知识库评测集` |
| 目标知识库 | 点击“确认”拉取列表后，通过“选 KB”下拉选择，不能留空 |
| Top K | `5` |
| 样本上限 | `3` |
| 同义问法 | 先关闭 |
| Embedding / Rerank 标签 | 用"选 KB"下拉选目标知识库,自动回填 |

点击“开始评测”后，页面会跳转到运行详情页并轮询状态。

## 8. 发起全量评测

小样本确认无误后，再跑全量：

| 字段 | 建议值 |
|---|---|
| Top K | `5` |
| 样本上限 | `0` |
| 同义问法 | 按需开启 |

说明：

- `样本上限 = 0` 表示不限制样本数量。
- 开启同义问法后，query 数会增加，耗时也会增加。
- 目标知识库必须通过“选 KB”下拉显式选择；如果列表不准确，请调整搜索条件后重新拉取。

## 9. 查看报告

评测完成后，页面详情会展示：

- `document_recall@5`
- `document_mrr`
- `empty_result_rate`
- `avg_latency_ms`
- 失败样本 Top N
- Markdown 报告
- 下载入口

报告文件落盘位置：

```text
reports/<run_id>/
```

每次运行会生成：

```text
manifest.json
summary.json
results.jsonl
results.csv
report.md
console.log
```

文件用途：

| 文件 | 用途 |
|---|---|
| `manifest.json` | 历史列表、运行状态、运行配置 |
| `summary.json` | 聚合指标 |
| `results.jsonl` | 每条 query 的详细召回结果 |
| `results.csv` | Excel 分析 |
| `report.md` | 人工复盘报告 |
| `console.log` | 运行日志 |

## 10. 常用 API 检查

健康检查：

```powershell
Invoke-RestMethod http://127.0.0.1:8200/api/health
```

评测集列表：

```powershell
Invoke-RestMethod http://127.0.0.1:8200/api/datasets
```

历史评测：

```powershell
Invoke-RestMethod "http://127.0.0.1:8200/api/runs?limit=20&offset=0"
```

查看某次运行：

```powershell
Invoke-RestMethod http://127.0.0.1:8200/api/runs/<run_id>
```

查看 Markdown 报告：

```powershell
Invoke-RestMethod http://127.0.0.1:8200/api/runs/<run_id>/report
```

删除一个评测集（主 JSONL + 草稿 + 审核元信息；删除前会自动生成一份带时间戳的备份）：

```powershell
Invoke-WebRequest -Method DELETE "http://127.0.0.1:8200/api/datasets/datasets%2Fgenerated%2F%E6%80%9D%E7%A7%91_Catalyst_1200_generated.jsonl"
```

`path` 段需要按 URL 段分别 `encodeURIComponent` 编码，Windows PowerShell 下 `Invoke-WebRequest` 会自动编码 `Invoke-RestMethod -Method DELETE` 不支持，可以用 `Invoke-WebRequest` 拿到 `Content` 后 `ConvertFrom-Json` 查看返回的 `backup_path`，便于误删后回滚。

## 11. 常见问题

### 12.1 页面一直是 mock 数据

确认前端启动前没有设置：

```powershell
$env:VITE_USE_MOCK="true"
```

如需切回真实后端，在当前 PowerShell 中执行：

```powershell
Remove-Item Env:VITE_USE_MOCK -ErrorAction SilentlyContinue
$env:VITE_API_BASE_URL="http://127.0.0.1:8200"
npm run dev
```

如果已经启动过前端，修改环境变量后需要停止 `npm run dev` 并重新启动。

### 12.2 前端请求失败

检查后端是否启动：

```powershell
Invoke-RestMethod http://127.0.0.1:8200/api/health
```

检查前端代理地址：

```powershell
$env:VITE_API_BASE_URL
```

应为：

```text
http://127.0.0.1:8200
```

### 12.3 Dify 连接失败

确认 Dify 服务已启动，并能访问：

```text
http://localhost/v1
```

如果 Dify 需要 token，请在页面 Dify API Key 中填写。

### 12.4 知识库列表没有目标项

处理方式：

- 在页面点击“加载知识库”或打开“选 KB”下拉，按当前 Dify API 地址和 Key 重新拉取知识库列表。
- 确认知识库名称、厂商、型号中包含 `华为` 和 `S1720`。
- 确认 Dify 的知识库列表接口可正常返回数据。

### 12.5 报告没有生成

检查运行详情页错误信息，或打开：

```text
reports/<run_id>/console.log
```

常见原因：

- Dify 未启动。
- Dataset ID 填错。
- 评测集路径不存在。
- 知识库尚未完成索引。

### 12.6 CSV 中文乱码

系统生成的 `results.csv` 会尽量使用 Excel 友好的编码。如果仍乱码，建议用 Excel 的“数据 -> 自文本/CSV”导入，并选择 `UTF-8`。

## 13. 推荐评测流程

1. 启动 Dify，并确认目标知识库已索引完成。
2. 启动 `Dify-KB-Eval` 后端。
3. 以真实模式启动前端。
4. 使用 `limit=3` 跑小样本。
5. 检查报告、失败样本和 `console.log`。
6. 使用 `limit=0` 跑全量。
7. 导出 `report.md` 和 `results.csv` 给团队复盘。
8. 如需跨版本对比，保留不同 `run_id` 的报告目录。
