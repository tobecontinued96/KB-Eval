# 评测集编辑器表格与增删查改体验优化设计

日期：2026-06-15

## 背景

当前评测集编辑器已经支持 JSONL 行级编辑、标签字段增删、删除二次确认、分页、本地校验、服务端校验、保存到草稿或主 JSONL，以及提交人工审核。截图中的主要问题集中在表格浏览态：长文本列和标签列容易把行高撑开，左侧基础字段出现大块空白，用户难以快速扫读样本。

用户已确认优化方向：表格浏览态要更紧凑，长内容放到详情区承载；同时优化增删查改功能。“查”采用 B 档能力：关键词搜索、场景/难度/厂商筛选、只看已修改、只看有错误、清空筛选和命中高亮。

## 目标

- 让样本表格在浏览态保持稳定行高，减少长文本导致的断层。
- 保留快速行内编辑能力，同时把长字段编辑迁移到更舒适的详情区。
- 优化新增、删除、查询、修改四类常用操作的入口、反馈和状态保持。
- 让筛选条件可组合，并能快速清空。
- 在搜索命中内容上提供轻量高亮，降低人工审核时定位成本。
- 不改变现有 JSONL 数据格式、保存接口和审核流程。

## 非目标

- 不引入批量编辑、多人审批、权限控制或完整工作台流程。
- 不改变后端接口契约。
- 不引入新的表格库或状态管理库。
- 不改变分页的基本模型，rows 仍在前端内存中，表格渲染当前页。
- 不重新设计整个页面视觉体系，只优化编辑器表格与工具栏区域。

## 设计方案

采用“紧凑表格 + 增强工具栏 + 详情区编辑”的方案。

表格负责快速扫读和短字段快速修改。每行高度稳定，长文本最多显示 2 行，标签列最多展示少量标签，剩余内容显示 `+N` 提示。点击行后打开或刷新行详情区，详情区展示完整主问题、评估关注点、期望文档、期望章节、期望关键词、同义问法和其他元数据，并承担长字段的主要编辑任务。

工具栏新增组合筛选能力：关键词、场景、难度、厂商、只看已修改、只看有错误。任一条件生效时显示“清空筛选”入口，并在结果计数中展示当前匹配行数。关键词命中在表格预览文本中高亮，命中范围覆盖 ID、厂商、型号、场景、主题、问题、评估关注点和列表字段。

新增行后，系统自动跳转到包含新行的页码，选中新行，并展开详情区，用户可以直接补全必填字段。删除行继续使用二次确认，但确认文案更明确；删除成功后选择相邻行，避免详情区悬空。修改字段后继续使用现有 dirty、modifiedRowSet、本地校验和服务端校验机制，并在行号区域和详情区顶部显示未保存状态。

## 交互细节

### 浏览表格

- 表头继续 sticky。
- 表格最小宽度可以略收窄，避免过度横向滚动。
- ID、厂商、型号、场景、主题、难度保持单行省略。
- 主问题和评估关注点显示 2 行预览，超出省略。
- 标签列展示前 2 到 3 个标签，超出显示 `+N`。
- 活跃行保留左侧蓝色强调线。
- 有错误的单元格保留红色状态，并在行号或详情区汇总错误数量。

### 新增

- “新增行”按钮仍在工具栏中。
- 点击后追加空行，设置 dirty，清空服务端错误。
- 自动进入新行所在页，设置 activeRowIndex 为新行。
- 详情区打开并提示必填字段，可直接编辑。

### 删除

- 操作列保留删除图标。
- 第一次点击进入确认态，显示“确认删除”和取消图标。
- 确认删除后移除当前行，清空服务端错误。
- 若删除的是活跃行，自动选中下一行；若没有下一行，则选中上一行；若表格为空，则关闭详情区。
- 若当前页因删除变空，页码回退到最后一个有效页。

### 查询

- 关键词搜索支持当前已有字段，并补充厂商、型号、评估关注点、同义问法。
- 场景筛选保留。
- 新增难度筛选和厂商筛选，选项从当前 rows 动态生成。
- 新增两个切换项：只看已修改、只看有错误。
- 条件变化时回到第 1 页。
- 展示“已筛选 X / 总计 Y 行”。
- 支持一键清空所有筛选条件。
- 搜索关键词在表格预览文本中高亮；为空或只使用下拉筛选时不高亮。

### 修改

- 短字段继续支持表格内直接修改。
- 主问题、评估关注点、列表字段在表格中保持轻量入口，详情区提供完整编辑。
- 列表字段在详情区可新增、编辑、删除；表格中仍保留快速添加和删除标签的能力。
- 修改后立即更新 rows、dirty、modifiedRowSet，并清空服务端错误。
- 保存、保存到草稿、提交审核流程保持现状。

## 组件边界

`DatasetEditor` 继续作为页面容器，负责 rows、筛选状态、分页、保存和审核动作。

`EditorRow` 负责紧凑行渲染、短字段编辑、标签预览、删除入口和行激活。

`RowDetailDrawer` 从只读回顾升级为行详情编辑区，复用现有 updateRow、addListItem、removeListItem、editListItem 逻辑，不直接保存文件。

可新增少量纯展示或小工具函数：

- `highlightText`：根据关键词渲染命中高亮。
- `buildFilterOptions`：从 rows 派生厂商、场景、难度选项。
- `rowMatchesFilters`：集中处理组合筛选规则。
- `nextActiveIndexAfterDelete`：计算删除后的相邻行焦点。

## 数据流

1. 页面加载 rows 与 baselineRows。
2. 用户设置关键词或筛选条件。
3. 前端根据 rows、filter、scenarioFilter、difficultyFilter、vendorFilter、modifiedOnly、errorOnly 得到 visibleRows。
4. 表格根据 page 和 pageSize 渲染 pagedRows。
5. 用户点击行，activeRowIndex 指向原始 rows 下标。
6. 用户在表格或详情区修改字段。
7. updateRow 更新 rows，dirty 置为 true，服务端错误清空，本地校验自动重算。
8. 用户保存或提交审核时沿用现有序列化、校验和 API 流程。

## 错误处理

- 本地校验错误仍由 localValidate 产生。
- 服务端校验错误仍写入 serverErrors。
- “只看有错误”同时匹配本地错误和服务端错误。
- 有错误行在行号区显示状态，详情区顶部列出该行错误摘要。
- 筛选后没有结果时，空状态提示用户清空筛选或调整条件。
- 删除后页码越界时自动回到最后有效页。

## 测试与验证

- 前端构建通过。
- 手工验证紧凑表格：长问题、长评估关注点和多标签不会撑出大块空白。
- 手工验证新增：新增后跳转到新行并打开详情区。
- 手工验证删除：删除当前行后选中相邻行，最后一行删除后详情区关闭。
- 手工验证查询：关键词、场景、难度、厂商、只看已修改、只看有错误可组合；清空筛选有效；命中高亮正确。
- 手工验证修改：表格和详情区修改都能触发未保存状态、本地校验和保存流程。
- 回归验证保存到草稿、保存到 JSONL、提交审核、查看 JSONL 和下载原文件不回退。

## 风险

- `DatasetEditor.tsx` 已经较长，继续堆叠逻辑会降低可维护性。实现时应抽出小型纯函数或局部组件，但不做大范围重构。
- 搜索高亮如果直接切割字符串，可能影响输入控件；只在预览文本中启用高亮。
- 详情区从只读变为可编辑后，表格与详情区需要共享同一行数据，必须避免局部草稿和 rows 不一致。
- 组合筛选增加状态后，分页和活跃行定位需要小心处理，避免选中不可见行造成困惑。
